# AGENTS.md

- 与工具/模型交互使用英语，与用户交互使用中文。
- 探索代码优先使用 ace-tool MCP。
- 使用 Maven 时追加 `-T 1.5C` 参数。
- 编码前先描述方案并等待批准；需求不明时先澄清。
- 代码完成后列出潜在问题并提供测试用例。
- 发现 bug 时先编写复现测试，再修复到测试通过。
- Git 提交或推送前，先检查当前仓库的 `git config user.name` 和 `git config user.email` 是否与目标 GitHub 账号一致。
- 若用户明确要求“不增加固定规则”，后续方案与实现不得再默认引入固定域名规则，应优先考虑动态推导或调用方显式提供的机制。
- 涉及较大逻辑改动或新增复杂代码时，必须为关键类型、核心函数、重要分支和不直观的 Go 并发 / 缓存 / I/O 逻辑补充中文注释，默认按 Go 初学者可读的标准编写。
